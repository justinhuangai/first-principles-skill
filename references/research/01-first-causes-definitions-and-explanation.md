# First Causes, Definitions, And Explanation

## Core Figures

Aristotle, Euclid, Richard Feynman

## Why This File Exists

Ground first-principles reasoning in first causes, clean definitions, explanatory depth, and the refusal to confuse elegant words with real understanding.

## Source Anchors

- Aristotle on first causes and explanatory knowledge
- Euclid on definitions, axioms, and primitives
- Feynman on understanding that survives translation and testing

## Distinctions That Matter

- definition versus conclusion
- primitive versus derived claim
- explanation versus redescription
- understanding versus verbal fluency

## Canonical Questions

- What is primitive here and what is inherited language?
- Which large noun needs definition before analysis?
- What would count as a real explanation rather than a relabeling?
- Can the claim survive a translation into plainer, more observable language?

## What This Changes In The Skill

- It forces big nouns to earn their keep through definition and decomposition.
- It sharpens the difference between explanation, relabeling, and rhetorical fluency.
- It gives the skill a cleaner starting point for abstract or overloaded questions.

## Good Moves

- ask for the first cause or primitive before following the narrative upward
- make overloaded concepts legible by splitting them into dimensions
- rewrite intellectual-sounding claims into statements that could fail
- prefer smaller clean definitions over grand totalizing labels

## Common Misuse

- treating category words as explanations
- stacking abstractions until the object disappears
- calling something fundamental merely because it sounds deep
- mistaking a beautiful phrase for a disciplined decomposition

## Example Translation

AGI is not a primitive. It is a bundle term. A first-principles move defines performance breadth, transfer, autonomy, reliability, cost, and deployment boundaries before asking about timelines.

## Import Into The Skill

Use this file when the user is asking about big nouns, root explanation, or the difference between knowing a word and understanding the system.

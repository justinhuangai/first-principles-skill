# Governance, Learning, And Viability

## Core Figures

Elinor Ostrom, Peter Senge, Chris Argyris, Stafford Beer

## Why This File Exists

Ground organizational governance and adaptation in shared-rule fit, learning loops, double-loop learning, and the viability of the control structure itself.

## Source Anchors

- Ostrom on common-pool governance and local rule fit
- Senge on learning systems and mental models
- Argyris on theories in use and double-loop learning
- Beer on viability and requisite variety

## Distinctions That Matter

- policy text versus real governance
- single-loop correction versus double-loop learning
- espoused values versus theories in use
- visibility versus true control capacity

## Canonical Questions

- What rules or norms actually govern behavior here?
- How does the system learn from mistakes beyond local correction?
- Where do declared values diverge from operating reality?
- Does the control structure have enough variety for the complexity it faces?

## What This Changes In The Skill

- It adds institutional and viability lenses to decisions that otherwise stay at product or team level.
- It improves reasoning about how systems learn, adapt, and remain governable over time.
- It helps the skill connect rules, feedback, and survival rather than treating them separately.

## Good Moves

- design governance around context, stewardship, and real correction loops
- surface the governing beliefs that keep reproducing failure
- separate learning rituals from learning mechanisms
- ask whether the system can remain viable under higher complexity without crushing the edge

## Common Misuse

- assuming governance problems are solved by writing more policy
- calling repeated error correction “learning” while the frame stays fixed
- taking declared cultural values literally when behavior says otherwise
- treating more central control as a universal answer to complexity

## Example Translation

A company may say it values experimentation, but if failure reliably produces blame, the real theory in use is “look correct in public.” First-principles reasoning has to work with the operating reality, not the stated aspiration.

## Import Into The Skill

Use this file when the problem is clearly about institutional fit, shared governance, recurring organizational learning failure, or long-run system viability.

## Notes

- Load this when the problem involves rule design, institutional drift, collective coordination, or long-run system viability.
- It is useful when the question is no longer just “what should we do” but also “how should the system keep functioning and learning.”

- Governance without learning turns into rigid policy.
- Learning without viable control structure becomes reflection without sustained coordination.

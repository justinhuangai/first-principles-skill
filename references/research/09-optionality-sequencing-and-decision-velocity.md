# Optionality, Sequencing, And Decision Velocity

## Core Figures

Jeff Bezos, Nassim Nicholas Taleb, Richard Rumelt

## Why This File Exists

Explain how reversibility, sequencing, strategic coherence, and decision velocity shape good action under uncertainty.

## Source Anchors

- Bezos on one-way versus two-way doors
- Taleb on optionality and asymmetric payoff structures
- Rumelt on diagnosis guiding coherent action

## Distinctions That Matter

- reversible versus irreversible choice
- speed versus recklessness
- option value versus premature commitment
- coherent action versus action volume

## Canonical Questions

- Is this a one-way or two-way door?
- What can be staged to buy learning before commitment?
- Where is optionality being created or destroyed?
- What diagnosis makes the chosen sequence coherent rather than random?

## What This Changes In The Skill

- It makes reversibility and sequencing explicit rather than treating all decisions as equal.
- It improves how the skill reasons about preserving upside while buying information.
- It keeps decision speed tied to option structure rather than temperament alone.

## Good Moves

- push fast on reversible calls and slow down on one-way commitments
- sequence learning before scale where possible
- treat optionality as a design property, not as indecision
- use diagnosis to prevent a pile of disconnected moves

## Common Misuse

- using “move fast” language on irreversible decisions
- keeping false optionality that only delays learning
- locking into scale before the core assumption is tested
- confusing many actions with strategic progress

## Example Translation

A product team may not need to decide the final architecture today. It may need to decide the cheapest sequence that keeps high-value future options alive while eliminating low-quality uncertainty fast.

## Import Into The Skill

Use this file when timing, staging, one-way doors, or strategic sequencing are central to the problem.

## Notes

- Load this when users face uncertain choices, portfolio questions, or sequencing problems with asymmetric upside or downside.
- It is especially useful when the right move is not one best answer but a better order of moves.

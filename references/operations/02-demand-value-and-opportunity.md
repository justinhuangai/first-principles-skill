# Demand, Value, And Opportunity

## Why This File Exists

This file combines demand-side opportunity scanning, user progress mapping, jobs to be done, value framing, and route-to-user thinking into a single operational module for finding where value is mis-served.

## Goal

Demand, Value, And Opportunity gives the skill a stronger, less fragmented operational lens for this class of problem.

## When This Lens Is Usually Right

- the user asks what opportunities still exist
- the product feels useful but value capture is weak
- the route from discovery to value realization is part of the problem
- the team is tempted to chase hot categories instead of painful jobs

## Core Questions

- What job is the user actually trying to get done?
- What progress is the user trying to make, and from what current struggle?
- Where is the current path still slow, expensive, risky, or humiliating?
- What value is created for the user versus merely described by the seller?
- What route gets this value to the user with the least fragility?

## Default Workflow

1. identify the user progress or relief that is actually being purchased
2. map the current path the user takes and where it breaks, slows, or feels wasteful
3. distinguish demand primitives from channels, features, or current category names
4. locate the step that becomes newly rewriteable because cost or capability changed
5. end with one opportunity statement tied to a real job, not a trend label

## Common Patterns

- hot market but weak job -> trend chasing without durable demand
- strong product but poor channel -> value exists but route-to-user is broken
- users tolerate bad workflow because switching costs are high -> latent opportunity
- buyers say price is the issue when time-to-value is the real blocker -> reframed value map
- category feels crowded because the real unmet progress is still poorly served -> opportunity under noise

## Good Moves

- define opportunities through underserved progress, not fashion
- tie value to observable consequence rather than marketing language
- separate demand quality from traffic quantity
- inspect both product fit and channel fit
- prefer opportunities where a step just became rewritable because capabilities got cheaper

## Failure Modes

- starting from “what is hot” instead of “what is painful”
- confusing user praise with durable willingness to pay
- treating all growth problems as messaging problems
- missing the fact that distribution is itself part of the product experience

## Natural Deliverables

- a job-to-be-done statement
- a user progress map
- a short list of underserved steps or pains
- a clearer route-to-user recommendation

## Example Translation

Question: "现在还有什么创业机会？"

A strong move is to stop asking which idea is hottest and instead inspect where users are still moving through a slow, expensive, frustrating path. The opportunity is usually a rewritten route to an old job, not a mystical new desire.

## High-Signal Heuristics

- A crowded category can still contain wide-open jobs.
- The best opportunities often sit in old demand with newly rewritable paths.
- Value language that cannot be translated into user state change is usually thin.
- Route-to-user problems often masquerade as product problems.

## Mini Cases

- A seemingly saturated software category still hides opportunity because onboarding time is painfully long for a neglected user segment.
- A product with strong praise fails because the buying route depends on a single brittle gatekeeper.
- A founder thinks the problem is weak demand when the real issue is that time-to-value is too slow for the intended buyer.

## Quick Checks

- What progress is the user hiring this for?
- Where is the current route still embarrassing, risky, or expensive?
- Is the bottleneck demand creation, channel fit, or value realization?

## Useful Prompts

- "What job is the user really hiring this product or service to do?"
- "Which painful step still exists only because the old path was expensive or awkward?"
- "What are users trying to avoid, not just what they say they want?"
- "Which part of the current solution is demand and which part is inherited packaging?"
- "What became newly possible because some input just got cheaper or faster?"
- "If we removed the current category label, what human progress remains?"
- "Where is willingness to pay strong even if the current market framing is weak?"
- "What opportunity only appears after we separate the demand from the delivery path?"

## Observation Checklist

- State the user’s desired progress in their own world, not in product language.
- Name the current workaround or patch users rely on today.
- Separate frequency from intensity; both matter differently.
- Check whether the opportunity depends on new capability or just better execution.
- Inspect where time, friction, embarrassment, or risk are currently being paid.
- Look for users who are overpaying with effort rather than cash.
- Avoid confusing audience size with urgency.
- Translate the opportunity into a simpler path to an existing demand.

## Red Flags

- The opportunity is described as a market trend rather than a human job.
- People are naming tools, not progress.
- The answer assumes demand because distribution suddenly got cheaper.
- A large TAM substitutes for a real pain statement.
- The path gets described from the company’s side, not the user’s side.
- Hotness is doing the work that evidence should do.
- The opportunity disappears if one buzzword is removed.

## Misreadings To Avoid

- Do not assume every painful process is worth a startup.
- Do not confuse a temporary arbitrage with durable demand.
- Do not over-index on stated preferences when costly behavior says otherwise.
- Do not treat “AI can do this now” as sufficient reason for existence.
- Do not flatten different user jobs into one synthetic market.
- Do not call every convenience improvement a category creation.

## Modern Questions This Helps With

- What startup opportunities appear when expert labor becomes partially automatable?
- Where is a B2B workflow still expensive because information is entered twice?
- Which creator or educator jobs are still solved through awkward manual packaging?
- What consumer demand remains under-served because trust costs stay high?
- Which SaaS-like categories are really service jobs waiting for redesign?
- Where does new distribution unlock a simpler path to an old high-friction need?

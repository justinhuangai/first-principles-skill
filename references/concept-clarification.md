# Concept Clarification

Use this mode when the user asks a question built on a large, overloaded, or abstract term.

## Goal

Turn a big word into cleaner primitives before analysis begins.

## When This Mode Is Usually Right

- the question asks for a date, strategy, or judgment before the object is defined
- two people may be using the same noun differently
- a large term is carrying too many unstated dimensions
- the user is asking about meaning, quality, intelligence, fit, value, or other compression-heavy concepts

## Output Pattern

1. Name the overloaded term
2. Define what should count and what should not count
3. Split it into dimensions, primitives, or necessary conditions
4. Surface the hidden assumptions bundled into the wording
5. Reframe the question into something that can guide judgment, design, or observation

## Workflow

1. Identify the term that is carrying too much meaning
2. Define what could count and what should not count
3. Split the term into dimensions, primitives, or necessary conditions
4. Surface which assumptions are bundled into the current wording
5. Reframe the question in a form that can guide judgment, design, or observation

## Useful Prompts

- "What exactly are we calling this?"
- "Which dimensions are being bundled together here?"
- "What would count as evidence that this is present?"
- "Are we arguing about reality, or just using different definitions?"

## Useful Distinctions

- definition disagreement vs reality disagreement
- capability vs deployment
- necessary condition vs sufficient condition
- label vs threshold

## Mini Example

Question:

- "When will AGI arrive?"

Clarification:

- define AGI in terms of breadth, transfer, autonomy, reliability, and acceptable cost
- separate "impressive demos" from "broad deployable competence"
- reframe from date prediction to bottleneck analysis

## Failure Modes

- pretending the noun already has one stable meaning
- asking for a timeline before defining the object
- turning abstract language into fake precision

## Boundary

Do not stay here forever. Once the term is clean enough, hand the problem to constraint decomposition, redesign, or experiment design.
